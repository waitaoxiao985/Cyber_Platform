# 外部访问配置规范

## 1. 需求分析

### 1.1 原始需求
- 当前网站只能通过 localhost:3000 在本地访问
- 目标：让所有人都能访问这个网站，而不仅仅是在主机上用 localhost 访问

### 1.2 技术背景
- 项目使用 Next.js 框架
- 使用 Prisma 作为 ORM
- 使用 NextAuth 进行身份验证
- 当前 NEXTAUTH_URL 配置为 http://localhost:3000

## 2. 实现方案

### 2.1 配置 Next.js 服务器
- 修改开发服务器配置，使其监听所有网络接口
- 确保生产环境服务器也能正确配置

### 2.2 环境变量配置
- 更新 NEXTAUTH_URL 环境变量，使其指向外部可访问的 URL
- 确保其他相关环境变量正确配置

### 2.3 网络和防火墙设置
- 确保防火墙允许外部访问 3000 端口（开发环境）或生产环境端口
- 如果在局域网内，可能需要设置端口转发

## 3. 实现步骤

### 3.1 开发环境配置
1. 修改 package.json 中的 dev 脚本，添加 --host 参数
2. 更新 .env 文件中的 NEXTAUTH_URL
3. 启动开发服务器并测试外部访问

### 3.2 生产环境配置
1. 构建项目：`npm run build`
2. 启动生产服务器，确保监听所有网络接口
3. 更新 NEXTAUTH_URL 环境变量
4. 测试外部访问

## 4. 验证步骤

### 4.1 本地验证
- 从本地浏览器访问网站，确保正常工作

### 4.2 网络验证
- 从同一网络中的其他设备访问网站
- 验证身份验证功能是否正常
- 验证所有页面和功能是否正常工作

### 4.3 外部网络验证（可选）
- 如果有公网 IP，从外部网络访问网站
- 验证所有功能是否正常

## 5. 安全考虑

### 5.1 注意事项
- 确保只开放必要的端口
- 考虑使用 HTTPS 协议
- 确保敏感信息不被泄露

### 5.2 建议
- 在生产环境中使用反向代理（如 Nginx）
- 配置适当的 CORS 策略
- 定期更新依赖包

## 6. 故障排除

### 6.1 常见问题
- 防火墙阻止访问
- 网络配置错误
- 环境变量配置错误

### 6.2 解决方案
- 检查防火墙设置
- 验证网络连接
- 确认环境变量正确配置

## 7. 验收标准

1. 网站可以通过局域网内的其他设备访问
2. 所有功能正常工作，包括身份验证
3. 网站响应速度正常
4. 没有安全漏洞

## 8. 后续维护

- 定期检查网络配置
- 监控服务器状态
- 及时更新依赖包和安全补丁
